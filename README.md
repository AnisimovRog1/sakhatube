# SakhaTube

Адаптивный интерфейс первой версии SakhaTube: карусель премьер, каталог сериалов, персональная лента «Для вас», загрузки, профиль и демонстрационный плеер.

## Запуск

Откройте `index.html` в браузере. Проект не требует установки зависимостей.

## Дальше

Интерфейс подготовлен как быстрый фронтенд-слой. Для рабочего сервиса далее подключаются: HLS/CDN, авторизация, каталог, платежи, серверная проверка покупок и веб-админка из продуктового фундамента.

## Studio

Отдельная [демо-админка](admin.html) показывает рабочие сценарии редакции: создание и удаление карточек, доступ и цена, очередь файлов, модерацию комментариев и сводные метрики. В разделе «Главная» есть живой предпросмотр телефона и desktop: баннер можно привязать к сериалу, изменить его текст, CTA, оформление, видимость и порядок. Изменения сохраняются только в браузере этого устройства. Файлы никуда не отправляются, а вход и роли не являются настоящими — публичную страницу нельзя использовать как боевую админку.

## Локальный API Starter

В `server/` появился отдельный каркас API Studio: роли, JWT-сессии, контент, порядок витрины, модерация, аудит и события плеера. Он предназначен только для локальной разработки: данные находятся в памяти процесса и исчезают после перезапуска. В production он намеренно откажется стартовать без `DATABASE_URL` и сильного `JWT_SECRET`.

```bash
npm install
npm run test:api
npm run dev:api
```

Админский интерфейс пока остаётся независимым демо, чтобы публичная GitHub Pages не выглядела как защищённая панель. Следующее подключение: закрытый домен Studio + OIDC/MFA + PostgreSQL + объектное хранилище + CDN.
