# SakhaTube

Адаптивный интерфейс первой версии SakhaTube: карусель премьер, каталог сериалов, персональная лента «Для вас», загрузки, профиль и демонстрационный плеер.

## Запуск

Откройте `index.html` в браузере. Проект не требует установки зависимостей.

## Дальше

Интерфейс подготовлен как быстрый фронтенд-слой. Для рабочего сервиса далее подключаются: HLS/CDN, авторизация, каталог, платежи, серверная проверка покупок и веб-админка из продуктового фундамента.

## Studio

Отдельная [Studio](admin.html) работает в двух безопасно разделённых режимах. Без сессии это явно обозначенный локальный предпросмотр: баннеры и тестовые карточки сохраняются только в браузере. После подключения короткоживущего staff-token Studio читает и меняет серверные контент, порядок главной, сводку и модерацию комментариев. Токен хранится только до закрытия вкладки. Загрузка многогигабайтных видео и баннеров через интерфейс ещё не подключена: не выдавайте Studio публичный доступ, пока не появятся OIDC/MFA, защищённая загрузка и видеоконвейер.

## Локальный API Starter

В `server/` есть API-основа Studio: роли, короткие JWT-сессии для разработки, контент, жизненный цикл публикации, юридическое подтверждение прав, порядок витрины, модерация, аудит, публичный каталог и события плеера. Публичные legal-страницы и формы поддержки/удаления данных доступны по `/privacy`, `/terms`, `/community-rules`, `/support`, `/delete-account`. В production сервис запускается только с PostgreSQL и сильным `JWT_SECRET`. Вход сотрудников через OIDC/MFA, права зрителей, платежи, защищённая видеозагрузка и HLS/DRM — следующие обязательные этапы, а не готовые функции.

```bash
npm install
npm run test:api
npm run dev:api
```

Публичную Studio нельзя считать закрытой админкой до подключения отдельного домена, OIDC/MFA и production-сессий. Следующий инфраструктурный этап: приватное объектное хранилище, защищённая многосоставная загрузка и видеоконвейер HLS/CMAF.
