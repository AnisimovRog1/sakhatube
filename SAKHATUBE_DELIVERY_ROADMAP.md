# SakhaTube — дорожная карта до запуска

Дата: 14 июля 2026. Документ отделяет уже готовую интерактивную демоверсию от настоящего продукта. Это не обещание «сделать всё сразу»: каждая фаза имеет измеримый результат, зависимости и критерий готовности.

## Принцип продукта

Зритель открывает приложение и начинает смотреть без регистрации; регистрация нужна только для синхронизации, покупки, сохранения или комментария. Главные маршруты телефона: **Главная · Каталог · Для вас · Офлайн · Профиль**. «Для вас» — персональная лента коротких видео и переходов в полные истории. Плеер открывается поверх них и не оставляет пользователя в тупике.

Профиль должен быть личной библиотекой, а не экраном с одной кнопкой:

- кто вошёл, язык и активный тариф;
- продолжить просмотр;
- история, избранное и «посмотреть позже»;
- загрузки и свободное место;
- уведомления с понятными настройками;
- безопасность, устройства, помощь и управление оплатой.

## Фаза 0 — рамки продукта и контента

**Цель:** не строить сервис до подтверждения прав, модели дохода и первой аудитории.

- [ ] Утвердить название, домен, юридическое лицо и политику конфиденциальности.
- [ ] Зафиксировать права на каждый первый сериал, клип, обложку, музыку, субтитры и территорию показа.
- [ ] Выбрать стартовый каталог: премьера, 3–5 сериалов, трейлеры, вертикальные клипы.
- [ ] Определить бесплатный предел, подписку, разовый доступ к сериалу и правила промокодов.
- [ ] Принять таблицу возрастных рейтингов, жалоб и редакционных ограничений.
- [ ] Описать один главный пользовательский путь: открыть → выбрать → начать смотреть → продолжить → купить при необходимости.
- [x] Подготовить независимый слой локализации в демоверсии: русский, English, саха тыла.
- [ ] Утвердить словарь интерфейса на русском и English; вычитать весь саха-текст редактором-носителем до публикации.

**Результат:** подписанный список прав, тарифов, правил показа и 10–15 реальных единиц стартового контента.

**Стоп-условие:** нельзя открывать платёж или публиковать эпизод без подтверждённых прав и владельца контента в админке.

## Фаза 1 — продуктовый прототип и UX-спецификация

**Цель:** проверить все важные маршруты до дорогой нативной разработки.

- [x] Главная с интерактивной каруселью премьер, каталог, «Для вас», офлайн и профиль в кликабельной демоверсии.
- [x] Быстрый поиск, фильтр жанров, карточки контента и модальное окно плеера.
- [x] Профиль: настройки имени, языка, автоперехода и экономии трафика сохраняются на устройстве демоверсии.
- [x] Уведомления, действия в клипах, переходы и состояние кнопок не являются пустыми элементами.
- [ ] Страница сериала: трейлер, описание, список эпизодов, доступ, возраст, субтитры, «продолжить».
- [ ] Экран покупки: цена, период, бесплатные эпизоды, восстановление покупок и условия отмены.
- [ ] Сценарии пустого состояния: нет сети, нет загрузок, нет результатов поиска, нет уведомлений.
- [ ] Сценарии ошибки: код входа, отказ оплаты, недоступная серия, истёкшая лицензия, медленная сеть.
- [ ] Проверка на телефонах шириной 320, 360, 375, 390 и 430 px; на планшете и desktop.
- [ ] Проверка с увеличенным системным размером текста и клавиатурой.
- [ ] Полная языковая QA: переполнения, переносы, субтитры и все системные состояния на русском, English и саха тыла.

**Результат:** Figma/прототип либо эта веб-демоверсия с картой всех экранов и сценариев. Ни один CTA не ведёт в никуда.

**Критерий готовности:** 5 тестовых пользователей без подсказки находят сериал, включают клип, меняют настройку профиля и понимают, где лежат загрузки.

## Фаза 2 — контентная админка (первая внутренняя версия)

**Цель:** редакция самостоятельно готовит и публикует каталог без разработчика.

- [x] Интерактивный прототип Studio: контент, порядок главной, очередь файлов, комментарии и метрики. Он хранит демо-данные только в браузере и не заменяет защищённый сервер.
- [ ] Авторизация сотрудников с MFA и ролями: superadmin, редактор, модератор, поддержка, финансы, аналитик.
- [ ] Сущности: сериал → сезон → эпизод; клип; трейлер; жанр; подборка; участник; лицензия.
- [ ] Черновик → проверка → запланировано → опубликовано → снято с показа.
- [ ] Поля контента: название, синопсис, обложка, рейтинг, языки, субтитры, дата, территория, доступ, цена.
- [ ] Загрузка мастер-файла, очередь транскодирования, статус обработки, проверка субтитров.
- [ ] Главная витрина: ручной порядок блоков, A/B-варианты только после появления аналитики.
- [ ] Аудит-лог: кто, что, когда и почему изменил; массовые операции только с подтверждением.

**Результат:** выпуск эпизода выполняется редактором через настоящую защищённую админку от загрузки до публикации.

**Критерий готовности:** тестовый эпизод опубликован и снят с показа без доступа к серверу и без потери аудита.

## Фаза 3 — серверное ядро и видеоконвейер

**Цель:** надёжно отдать легальное видео на телефон и не смешивать видео, оплату и админку в одном сервисе.

- [x] Локальный API Starter: защищённые ролями маршруты контента, витрины и модерации, аудит действий и приём событий плеера. Временное in-memory хранилище предназначено только для тестов.
- [ ] API gateway, каталог, аккаунт, права доступа, история, избранное, загрузки, поиск и уведомления.
- [ ] PostgreSQL для транзакционных данных; Redis для кэша и ограничений; объектное хранилище для исходников и HLS.
- [ ] Конвейер: проверка файла → транскодирование → HLS/CMAF → субтитры → подпись манифеста → CDN.
- [ ] Несколько профилей качества: старт с безопасного качества, дальнейший адаптивный выбор по сети.
- [ ] Подписанные короткоживущие URL и серверная проверка права на каждый платный поток.
- [ ] Очередь задач с повтором: транскодирование, письмо/пуш, синхронизация оплаты, аналитика.
- [ ] Наблюдаемость: структурированные логи, трассировка, метрики первого кадра, буферизации, ошибок и CDN.
- [ ] Резервные копии, тест восстановления и изоляция production от тестовых сред.

**Результат:** один эпизод можно легально отдать на тестовые iOS/Android устройства через CDN с адаптивным качеством.

**Критерий готовности:** при плохой сети видео не зависает бесконечно: качество снижается либо пользователь видит ясную повторную попытку.

## Фаза 4 — нативные приложения и настоящий плеер

**Цель:** сделать скорость и ощущение качества, которые нельзя обеспечить статической веб-демоверсией.

### iOS

- [ ] Swift + SwiftUI для экранов; AVPlayer/AVFoundation для HLS.
- [ ] Keychain для сессии, фоновые загрузки, Picture-in-Picture по правилам платформы.
- [ ] VoiceOver, Dynamic Type, безопасные зоны, iPhone и iPad.

### Android

- [ ] Kotlin + Jetpack Compose; Media3/ExoPlayer для HLS.
- [ ] Keystore, WorkManager для очередей, адаптация под телефоны/планшеты/складные устройства.
- [ ] Baseline Profile и Macrobenchmark для холодного старта и первых экранов.

### Общий плеерный контракт

- [ ] «Продолжить с 00:43», следующая серия, ручное качество, скорость, субтитры, автоплей, блокировка поворота.
- [ ] Предзагрузка следующего манифеста и первого сегмента; никакого чёрного экрана между сериями.
- [ ] Встроенные состояния: загрузка, плохая сеть, запрещённый доступ, субтитры не найдены, офлайн-лицензия истекла.
- [ ] Встроенная аналитика воспроизведения: запрос → первый кадр → пауза → буферизация → ошибка → завершение.

**Результат:** beta-версии iOS и Android с одним настоящим сериалом, HLS-плеером и гостевым просмотром.

**Критерий готовности:** p95 первого кадра на согласованной 4G-сети ≤ 2.0 с, crash-free users ≥ 99.8%, повторная буферизация < 0.5% сессий до выхода из beta.

## Фаза 5 — аккаунт, доступ и деньги

**Цель:** пользователь не теряет доступ, а деньги и права невозможно «нарисовать» в интерфейсе.

- [ ] Гостевой режим; вход через Apple/Google/телефон с лимитами на SMS и понятным резервным сценарием.
- [ ] Профиль и библиотека синхронизируются между устройствами после входа.
- [ ] Подписки и разовые покупки через App Store / Google Play; серверная валидация и вебхуки магазинов.
- [ ] Моментальное обновление entitlement после оплаты, «Восстановить покупки», журнал статуса и безопасная поддержка.
- [ ] Ограничение числа устройств и интерфейс управления активными сеансами.
- [ ] Возрастные профили и ограничения показа, если в продукте будет семейный сценарий.
- [ ] Чёткие экраны отмены, возврата и обращения в поддержку — без скрытых условий.

**Результат:** тестовая покупка открывает платный эпизод на двух устройствах одного пользователя и восстанавливается после переустановки.

**Критерий готовности:** 99.9% подтверждённых покупок получают доступ не позднее 10 секунд; для любой ошибки есть видимый статус и ID обращения.

## Фаза 6 — клипы, библиотека и рекомендации

**Цель:** сделать короткий формат каналом открытия сериалов, не превращая основное приложение в бесконечную ленту.

- [ ] Вертикальный HLS-клип 9:16, свайп, предзагрузка ровно следующего клипа.
- [ ] У каждого клипа есть источник: сериал, эпизод, тайм-код и кнопка «смотреть целиком».
- [ ] Лайк, сохранить, поделиться, «не интересно», жалоба; все события записываются серверно после согласия пользователя.
- [ ] Отдельная библиотека: история, избранное, «позже», продолжить и загрузки.
- [ ] Рекомендации сначала правилами: жанр, сериал, редакторские блоки, продолжение просмотра; затем безопасная персонализация.
- [ ] Объяснение рекомендации и возможность убрать нежелательный контент.
- [ ] Поиск с опечатками, актёрами, жанрами, тегами и быстрыми фильтрами.

**Результат:** клип ведёт к полной истории, а сохранённый контент появляется в профиле на другом устройстве.

**Критерий готовности:** клипы не ухудшают плеер серий; пользователь может полностью отключить автоплей клипов и рекомендации.

## Фаза 7 — уведомления, поддержка и доверие

**Цель:** уведомления возвращают к желанному контенту, а не раздражают.

- [ ] Центр уведомлений внутри приложения: новые серии, готовые загрузки, статус оплаты, ответ поддержки.
- [ ] Настройки по типам: новинки, продолжение, загрузки, сервисные; выключатель всех push-уведомлений.
- [ ] Ограничение частоты и дедупликация; не слать несколько одинаковых пушей при массовом релизе.
- [ ] Поддержка: обращение, категория, вложение, безопасный ID заказа, статусы и SLA.
- [ ] Страница статуса сервиса и готовый процесс инцидентов.

**Результат:** колокольчик не является декорацией: он ведёт во входящие, а каждый тип можно отключить без звонка в поддержку.

## Фаза 8 — безопасность, защита контента и качество

**Цель:** снизить риск утечки видео, денег и персональных данных; быстро обнаружить проблему.

- [ ] OWASP MASVS/MASTG для iOS/Android и OWASP ASVS для серверной части.
- [ ] OIDC/OAuth, короткие токены, Keychain/Keystore, MFA сотрудников, RBAC, неизменяемый аудит-лог.
- [ ] WAF, rate limit, защита SMS/паролей от перебора, проверка входящих вебхуков, секреты в KMS.
- [ ] DRM и/или подписанные потоки для платного контента; watermark для дорогих премьер только при законном основании.
- [ ] Шифрование в транзите и на диске, минимизация персональных данных, проверка резервных копий.
- [ ] Нагрузочное тестирование, хаос-тест отказа CDN/БД, пентест до публичного запуска.
- [ ] Мониторинг из реальных целевых сетей: запуск, каталог, первый кадр, вход, покупка, восстановление доступа.

**Результат:** отчёт threat model, пентест с закрытыми критичными замечаниями, runbook для инцидента.

## Фаза 9 — закрытая beta и запуск

**Цель:** найти реальные проблемы до широкого трафика.

- [ ] Закрытая группа iOS TestFlight и Google Play testing с реальными устройствами и разными сетями.
- [ ] Еженедельный разбор: старт приложения, первый кадр, буферизация, падения, оплата, возврат, поиск, загрузки.
- [ ] Поддержка «красной кнопки» для случаев оплаты без доступа и недоступного плеера.
- [ ] Набор витринных материалов, возрастные рейтинги, App Store / Google Play privacy labels и политика.
- [ ] План релиза по процентам аудитории, быстрый rollback и feature flags для опасных функций.
- [ ] До запуска: контентный календарь, промо-клипы, редакторские подборки и понятная страница подписки.

**Результат:** публичный запуск только после прохождения метрик качества и отсутствия блокирующих ошибок оплаты/видео.

## Фаза 10 — рост после стабильного запуска

- [ ] Creator Studio: загрузка, субтитры, права, аналитика, комментарии, монетизация — desktop-first.
- [ ] Smart TV после стабилизации мобильного плеера; управление пультом и привязка через QR.
- [ ] Live, рекламный слой между эпизодами и партнёрские витрины — только после контроля частоты и прав.
- [ ] Эксперименты с рекомендациями и ценами через feature flags с защитой от ухудшения пользовательского опыта.
- [ ] Bug bounty и регулярные внешние проверки безопасности.

## Что делать следующим рабочим спринтом

1. Утвердить первый каталог и права (фаза 0).
2. Закрыть UX-карту: страница сериала, покупка, ошибки и пустые состояния (фаза 1).
3. Выбрать облако/CDN и запустить один реальный HLS-эпизод (фаза 3).
4. Начать iOS/Android приложения параллельно с минимальной админкой контента (фазы 2 и 4).
5. Не подключать платёж до серверного entitlement-сервиса и восстановления покупок (фаза 5).

## Контроль качества интерфейса на всех фазах

- Каждая кнопка имеет действие, состояние загрузки и ясный результат.
- На телефоне любой интерактивный элемент находится в зоне нажатия не меньше 44×44 pt / dp.
- Есть проверка на пять мобильных ширин, безопасные зоны, крупный текст, русские длинные строки и тёмную тему.
- Экран не должен зависеть от декоративного изображения: контент, заголовок и основное действие остаются понятны без него.
- Перед merge — визуальная проверка экранов на эталонных размерах; перед release — автоматические screenshot-тесты и тесты главных маршрутов.
