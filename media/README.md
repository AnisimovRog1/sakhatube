# Легальный тестовый каталог SakhaTube

Эта папка описывает временные ролики для проверки загрузки, HLS-плеера и вертикальной ленты. В неё нельзя добавлять видео, скачанные у авторов с YouTube или других площадок без письменного разрешения/лицензии.

`demo-catalog.json` содержит источник, лицензию и обязательную атрибуцию для каждого исходного ролика. Временные исходники и подготовленные HLS-файлы находятся в `media/source/` и `media/hls/`; они намеренно исключены из Git.

## Набор «витрина в действии»

`npm run media:create-cc-test-set` создаёт из уже подготовленных CC-исходников небольшую витрину без новых скачиваний: 5 горизонтальных карточек, из которых `Sintel` — один мини-сериал из 3 эпизодов, и 10 настоящих вертикальных роликов 1080×1920. Он сохраняет в `media/hls/cc-test-set/catalog.json` и `credits.json` обязательную атрибуцию для каждой производной записи.

Два исходника из `demo-catalog.json` занимают около 1,78 GB суммарно — это ниже предела 5 GB, установленного загрузчиком. Производные MP4 и обложки создаются локально и не входят в Git. Набор предназначен только для закрытого тестирования: перед публичным запуском удалить его из витрины и Bucket или заменить собственным лицензированным контентом.

## Локальный конвейер

```bash
npm run media:download-demo
npm run media:package-demo
CONFIRM_DEMO_UPLOAD=yes npm run media:upload-demo
```

Для витрины из 5 карточек, мини-сериала и 10 Shorts после подготовки двух исходников:

```bash
npm run media:create-cc-test-set
CONFIRM_DEMO_UPLOAD=yes npm run media:upload-cc-test-set
```

Для загрузки в Railway Bucket нужны переменные окружения `S3_ENDPOINT`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY`, `S3_BUCKET` и `S3_REGION`.

Пока это тестовый каталог. Перед публичным запуском его нужно удалить из витрины вместе с HLS-файлами либо заменить материалами, на которые SakhaTube владеет правами.
