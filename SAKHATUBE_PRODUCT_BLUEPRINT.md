# SakhaTube — продуктовый фундамент

Дата: 14 июля 2026. Это самостоятельный продукт, созданный по изученным открытым практикам видеосервисов; не копия QUBE.

## 1. Что нужно исправить относительно QUBE

Публичные отзывы QUBE указывают на три критичные точки: не приходит код регистрации, видео долго стартует/зависает, покупка не всегда открывает доступ. Для SakhaTube это обязательные стоп-условия качества:

1. Вход: телефон с резервным входом через Apple/Google; понятная ошибка и повторная отправка кода.
2. Видео: адаптивный HLS, CDN, предзагрузка следующей серии и автоматическое снижение качества при плохой сети.
3. Покупка: серверная проверка оплаты, мгновенное обновление прав доступа, экран «Восстановить покупки», журнал платежа для поддержки.

## 2. Продукт

SakhaTube — легальная мобильная видеоплатформа сериалов, шоу и коротких вертикальных роликов. Платформа должна открываться без VPN в целевых сетях: контент, API и мониторинг размещаются у легальных провайдеров, доступных пользователям. Не используются обходы блокировок.

### Принципы

- Открыть приложение → начать смотреть за секунды, без обязательной регистрации.
- Одно основное действие на экране: смотреть, продолжить, купить или скачать.
- Цена и доступ показываются до оплаты. Никаких скрытых списаний и ложных таймеров.
- Платный контент защищён, но пользователь всегда видит, почему серия закрыта и как открыть её.
- Рекомендации объяснимы: «потому что вы смотрели…», с кнопкой «не интересно».

| Экран | Телефон | Планшет | Веб/TV |
|---|---|---|---|
| Старт / язык | 1 экран, саха/русский | тот же сценарий, широкие отступы | не обязателен на TV |
| Главная | вертикальная лента: продолжить, премьеры, подборки | лента + боковая навигация | витрина из крупных карточек |
| Каталог / поиск | поиск, жанры, фильтры, карточки 2 колонки | 3–5 колонок, фильтры слева | 5+ колонок, управление с пульта на TV |
| Страница сериала | обложка, описание, прогресс, эпизоды | обложка и эпизоды рядом | горизонтальная серия карточек |
| Плеер | нативный вертикальный фуллскрин, свайп следующая серия | плеер + список эпизодов | плеер 16:9/вертикальный режим по контенту |
| Платный доступ | прозрачная цена, пакет, подписка, восстановление покупок | тот же поток без модального хаоса | QR/привязка к мобильному платежу при необходимости |

| Админка | только срочные алерты | рабочий режим | основной формат: desktop-first |

### Навигация телефона

`Главная · Каталог · Клипы · Офлайн · Профиль` — пять постоянных пунктов. Поиск открывается из шапки и не занимает отдельную вкладку. Плеер скрывает навигацию. Кнопка «назад» всегда возвращает в предыдущее логичное состояние, а не в начало.

## 4. Плеер — главный экран

Обязательные функции:

- HLS с несколькими качествами, автоподбором по сети и быстрым переключением качества.
- Старт с минимального безопасного качества, затем повышение; предзагрузка манифеста и первого сегмента следующего эпизода.
- «Продолжить с 00:43», переход к следующей серии без чёрного экрана.
- Ручной выбор качества, субтитры на саха/русском, скорость 0.75–2x, блокировка поворота.
- Кнопки: назад, пауза, эпизоды, субтитры, качество, избранное, пожаловаться. Никакого слоя из лишних кнопок.
- Офлайн: скачать лицензированный HLS-контент в защищённое хранилище приложения; показывать срок действия и объём.

## 5. Монетизация без раздражения

1. Бесплатно: трейлер и первые 2–3 эпизода каждого сериала.
2. Подписка: полный каталог, загрузки, без рекламы.
3. Пакеты: разовый доступ к отдельному сериалу для тех, кому не нужна подписка.
4. Промокоды: только через официальный кабинет и с журналом применения.
5. Реклама: только в бесплатном слое и только между эпизодами; никогда не разрывает сцену.

Права доступа рассчитываются на сервере, а не в интерфейсе приложения. Любая покупка проверяется через серверные уведомления магазинов и сохраняется в неизменяемом журнале.

## 5.1 Клипы — короткие нарезки

Отдельный раздел «Клипы» нужен для тизеров, лучших сцен, коротких интервью и авторских вертикальных роликов. Он не смешивается с полными эпизодами:

- Вертикальное видео 9:16, свайп/стрелки к следующему клипу.
- У каждого клипа есть источник: сериал, эпизод и тайм-код; переход ведёт на полную историю.
- Кнопки только по делу: нравится, сохранить, поделиться, пожаловаться.
- Предзагружается только следующий клип; при плохой сети качество снижается автоматически.
- Полный сериал открывается в отдельном плеере и не прерывается рекламой внутри сцены.

## 6. Админка: центральный пункт управления

| Раздел | Контроль |
|---|---|
| Главная | выручка, активные зрители, ошибки плеера, жалобы, непроведённые платежи |
| Контент | сериалы, сезоны, эпизоды, обложки, локализации, субтитры, статусы |
| Публикация | черновик → проверка → запланировано → опубликовано → снято с показа |
| Права | страны, даты лицензий, тарифы, доступность, возрастной рейтинг |
| Платежи | покупки, возвраты, восстановление доступа, промокоды, антифрод-метки |
| Модерация | жалобы, комментарии, блокировки, возрастные ограничения, решение и причина |
| Поддержка | карточка пользователя, история действий, платежи, безопасная выдача доступа |
| Рекомендации | редакторские подборки, правила исключения, A/B-тесты только с безопасными границами |
| Качество видео | время первого кадра, буферизация, ошибки CDN, устройства и регионы |
| Роли и аудит | кто что изменил, когда и почему; экспорт журнала только для уполномоченных |

### Роли

- `superadmin`: минимум людей; управление ролями и критичными настройками.
- `content_editor`: каталог и публикации, без платежей и ролей.
- `moderator`: жалобы и комментарии, без удаления финансовых данных.
- `support`: просмотр ограниченных данных, возврат/восстановление только по разрешению.
- `finance`: платежи и отчёты, без контента.
- `analyst`: обезличенные показатели.

Каждому действию в админке назначается роль, причина и аудит-лог. Для сотрудников обязательны MFA и отдельные рабочие учётные записи.

## 7. Техническая архитектура

```text
iOS: Swift + AVFoundation/AVPlayer       Android: Kotlin + Media3/ExoPlayer
                 │                                   │
                 └────────── API gateway ────────────┘
                              │
       Auth · каталог · права доступа · платежи · рекомендации · админка
                              │
   PostgreSQL · Redis · поисковый индекс · объектное хранилище · очередь задач
                              │
Загрузка → проверка → транскодирование → HLS/CMAF → DRM/подписанные URL → CDN
```

- Нативные клиенты выбраны ради стабильного плеера и офлайн-функций.
- API и админка отделены от видеораздачи: пик просмотров не должен останавливать платежи и управление контентом.
- Видео хранится как исходник + транскодированные варианты; пользователю не отдаётся исходный файл.
- Используются наблюдаемость и реальные проверки из ключевых целевых сетей: запуск, каталог, первый кадр, покупка, восстановление доступа.
- Перед запуском: нагрузочные тесты, тест на медленной сети, отказ одного узла/CDN, восстановление базы, проверка платежных вебхуков.

## 8. Защита

Абсолютно «невзламываемых» систем не существует. Цель — сделать атаку дорогой, быстро обнаруживаемой и без возможности массово украсть видео, деньги или персональные данные.

- Защита приложения: безопасное хранение токенов в Keychain/Keystore, короткоживущие сессии, MFA для чувствительных операций, защита от подмены приложения и корневых/jailbreak-устройств как дополнительный сигнал риска.
- Защита API: OAuth/OIDC, проверка прав на каждом запросе, rate limit, защита от перебора SMS-кодов, WAF, DDoS-защита, валидация всех входных данных.
- Защита видео: HLS с DRM для платного каталога, подписанные короткоживущие ссылки, контроль одновременных устройств, видимый/невидимый watermark для дорогих премьер.
- Защита данных: шифрование в транзите и на диске, секреты в KMS, сегментация сетей, резервные копии с тестом восстановления, минимизация собираемых данных.
- Защита админки: MFA, RBAC, IP/устройственные политики при необходимости, аудит-лог без возможности правки, принцип двух подтверждений для массовых возвратов и удаления контента.
- Процесс: независимый pentest до релиза, регулярное исправление уязвимостей, bug bounty после стабилизации, план реагирования на инциденты.

Стандарт проверки: OWASP MASVS/MASTG для мобильных приложений и OWASP ASVS для серверной части.

## 9. Дизайн и психология без манипуляций

Нет универсального «цвета, который заставит людей платить». Результат дают скорость, читаемость, предсказуемая навигация, локальная идентичность и честный доступ к контенту. Цвет не используется как уловка.

- База: тёмный кинематографичный интерфейс, чтобы обложки и видео были главным объектом.
- Акцент: один тёплый северный цвет для основного действия и один холодный для системных состояний; не более одного конкурирующего CTA на экране.
- Типографика: крупные названия, короткие описания, явный возрастной рейтинг и цена рядом с кнопкой доступа.
- Доступность: контраст текста не ниже WCAG AA 4.5:1, системный размер текста, подписи и субтитры, отсутствие агрессивных вспышек.
- Локальный характер строится на оригинальном контенте, языке саха, авторах и обложках — не на декоративных «этнических» эффектах.

## 11. Порядок разработки

1. Основа: дизайн-система, iOS/Android плеер, HLS/CDN, каталог, гостевой режим, админка контента.
2. Деньги и доверие: авторизация, подписка/пакеты, серверная проверка покупок, поддержка, права на контент, модерация.
3. Рост: рекомендации, поиск, загрузки, субтитры, Creator Studio, отчёты качества.
4. Масштаб: Smart TV, live, рекламный слой, партнёрская аналитика, watermark и расширенный DRM.
