# SakhaTube iOS

Нативная основа приложения для iPhone и iPad на SwiftUI. Это не WebView: каталог получает публичные данные по API, а интерфейс и будущий плеер работают нативно.

## Запуск

1. Откройте [SakhaTube.xcodeproj](SakhaTube.xcodeproj) в Xcode.
2. Выберите iPhone Simulator.
3. Запустите локальный SakhaTube API из корня репозитория.
4. Нажмите ▶ в Xcode.

В Debug приложение запрашивает `http://127.0.0.1:3000` — это адрес сервера на Mac, доступный из Simulator. Release использует HTTPS production API. На экранах «Главная» и «Каталог» приложение использует только публичные маршруты:

- `GET /v1/catalog/home`
- `GET /v1/catalog`

На устройстве iPhone `127.0.0.1` означает сам телефон, поэтому перед запуском нужно указать доступный устройству HTTPS-адрес API.

## Как поменять адрес API

Адрес не зашит в Swift-код. В Xcode откройте target **SakhaTube** → **Build Settings**, найдите `SAKHATUBE_API_BASE_URL` и укажите нужный URL без завершающего `/`.

Для развёрнутой тестовой среды можно указать:

`https://sakhatube-production.up.railway.app`

В релизе всегда используйте свой HTTPS-домен. Настройка локальной сети включена только для разработки на Simulator / в локальной сети и не заменяет защищённый production API.

## Демо-данные

Если сервер недоступен, на главной и в каталоге есть явная кнопка **«Открыть демо-каталог»**. Она показывает локальные fixtures и не смешивает их с серверными данными.

Для Xcode Previews данные включаются автоматически. Для офлайн-разработки в схеме Run добавьте аргумент запуска:

`-useDemoCatalog`

## Что уже реально подключено

- Codable-модели публичного каталога;
- загрузка главной витрины и каталога;
- фильтрация: в приложение попадают только записи со статусом `published`;
- загрузочные, пустые и сетевые error-состояния;
- ручный переход на локальный демо-каталог;
- конфигурируемый адрес API через `Info.plist` / Build Settings.

## Текущие ограничения

- нет входа, профиля на сервере, синхронизации истории и избранного;
- нет защищённого воспроизведения, выдачи прав, подписок или покупок;
- API пока не передаёт реальные постеры, сезоны, эпизоды и защищённые HLS-манифесты — интерфейс использует безопасные визуальные значения;
- демонстрационный AVPlayer не является финальным защищённым видеоплеером;
- добавлены App Icon и Privacy Manifest, но ещё нет подписывания команды, TestFlight-архива, accessibility/UITest-покрытия и production-плеера.

Перед TestFlight необходимо заменить bundle ID `com.sakhatube.app` на зарегистрированный в Apple Developer, настроить подпись, права на контент, политику конфиденциальности, удаление аккаунта и реальные серверные покупки.
